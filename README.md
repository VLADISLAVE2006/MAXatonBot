# MAXatonBot

Бот для записи абитуриентов на мероприятия МИРЭА в мессенджере [MAX](https://max.ru).  
Неофициальный студенческий проект, не аффилирован с университетом.

## Стек

| Компонент    | Технологии                                         |
| ------------ | -------------------------------------------------- |
| **backend**  | Go 1.26, Gorilla Mux, PostgreSQL, JWT (HS256)      |
| **bot**      | TypeScript, `@maxhub/max-bot-api`, node-cron, pnpm |
| **frontend** | React, Vite, SCSS                                  |

## Архитектура

```
┌─────────────┐        ┌─────────────┐        ┌─────────────┐
│   Frontend  │        │     Bot     │        │   Backend   │
│  (React)    │◄──────►│ (MAX Bot)   │◄──────►│  (Go REST)  │
│  Абитуриент │        │             │        │             │
│  Организатор│        │  Reminders  │        │  PostgreSQL │
└─────────────┘        └─────────────┘        └─────────────┘
       │                      │                      │
       │  deeplink ──────────►│                      │
       │  ?start=register_{id}│                      │
       │                      │  X-API-Key + JWT      │
       └──── VITE_API_URL ─────────────────────────►│
```

**Взаимодействие:**

- Фронтенд — витрина мероприятий для абитуриентов и панель организатора. Кнопка «Записаться» открывает бота через deeplink.
- Бот — основная точка входа пользователей. Общается с бэкендом через REST API с двойной аутентификацией: `X-API-Key` (между сервисами) + `Bearer JWT` (от имени пользователя).
- Бэкенд — единственный источник данных. Также отправляет вебхук-уведомления боту при изменении мероприятий.

## Роли

| Роль        | Возможности                                                                   |
| ----------- | ----------------------------------------------------------------------------- |
| `applicant` | Просмотр мероприятий, запись/отмена, история, QR-код, отзывы                  |
| `organizer` | Создание и редактирование мероприятий, загрузка CSV, статистика, посещаемость |
| `admin`     | Создание организаторов, загрузка соглашений об обработке ПД                   |

## Быстрый старт

### Требования

- Go 1.22+
- Node.js 20+ и pnpm
- PostgreSQL 14+

### 1. Backend

```bash
cd backend
cp .env.example .env   # заполнить переменные
go run ./main.go
```

**Переменные окружения:**

| Переменная        | Описание                                                        |
| ----------------- | --------------------------------------------------------------- |
| `DB_HOST`         | Хост PostgreSQL                                                 |
| `DB_PORT`         | Порт PostgreSQL (обычно `5432`)                                 |
| `DB_USER`         | Пользователь БД                                                 |
| `DB_PASSWORD`     | Пароль БД                                                       |
| `DB_NAME`         | Имя базы данных                                                 |
| `API_KEY`         | Общий секрет между backend и bot                                |
| `PORT`            | Порт HTTP-сервера (по умолчанию `8080`)                         |
| `JWT_SECRET`      | Секрет для подписи JWT-токенов                                  |
| `ADMIN_USER_ID`   | User ID администратора в MAX (выдаётся роль `admin` при старте) |
| `BOT_WEBHOOK_URL` | URL вебхука бота для уведомлений об изменении мероприятий       |

### 2. Bot

```bash
cd bot
cp .env.example .env   # заполнить переменные
pnpm install
pnpm dev               # режим разработки с watch
```

**Переменные окружения:**

| Переменная    | Описание                                                       |
| ------------- | -------------------------------------------------------------- |
| `BOT_TOKEN`   | Токен бота из кабинета MAX                                     |
| `ADMIN_ID`    | User ID администратора (совпадает с `ADMIN_USER_ID` в backend) |
| `API_KEY`     | Общий секрет (совпадает с `API_KEY` в backend)                 |
| `API_URL`     | Базовый URL backend, например `http://localhost:8080`          |
| `CATALOG_URL` | URL фронтенда-каталога мероприятий                             |
| `BOT_LINK`    | Прямая ссылка на бота, например `https://max.ru/bot_username`  |

### 3. Frontend

```bash
cd frontend
cp .env.example .env   # заполнить переменные
pnpm install
pnpm dev
```

**Переменные окружения:**

| Переменная          | Описание                            |
| ------------------- | ----------------------------------- |
| `VITE_API_URL`      | Базовый URL backend                 |
| `VITE_APP_URL`      | Публичный URL самого фронтенда      |
| `VITE_BOT_USERNAME` | Username бота (для deeplink-кнопок) |
| `VITE_API_KEY`      | `API_KEY` для запросов к backend    |

Фронтенд имеет две точки входа: `main.jsx` (каталог для абитуриентов) и `main-organizer.jsx` (панель организатора). В `vite.config.js` они настроены как отдельные MPA-страницы.

## Структура проекта

```
MAXatonBot/
├── backend/
│   ├── internal/
│   │   ├── db/          — инициализация и подключение к PostgreSQL
│   │   ├── handlers/    — HTTP-обработчики (events, user, agreement, webhook)
│   │   └── middleware/  — аутентификация (APIAuth, UserAuth, OrganizerAuth, AdminAuth)
│   └── main.go          — роутер и точка входа
├── bot/
│   └── src/
│       ├── commands/    — обработчики кнопок и команд (events, my_events, menu, ...)
│       ├── flows/       — пошаговые сценарии (registration, review, admin, csv_upload)
│       ├── utils/       — вспомогательные функции
│       ├── api.ts       — клиент к backend REST API
│       ├── context.ts   — сессии пользователей и AppContext
│       ├── reminders.ts — cron-напоминания (за день и за час)
│       └── webhook.ts   — приём вебхуков от backend
└── frontend/
    └── src/
        ├── components/  — EventCard, EventModal, FilterPanel, CreateEventModal, ...
        ├── api.ts       — клиент к backend REST API
        ├── App.jsx      — SPA для абитуриентов
        └── OrganizerApp.jsx — SPA для организаторов
```

## Напоминания

Бот автоматически отправляет напоминания записавшимся пользователям:

- **За день** — 07:00 по московскому времени
- **За час** — каждые 30 минут проверяет предстоящие мероприятия

Работает через `node-cron`, конфигурируется в [bot/src/reminders.ts](bot/src/reminders.ts).

## Загрузка мероприятий через CSV

Организатор может загрузить несколько мероприятий одним файлом прямо в боте.  
Формат CSV описан в [bot/src/flows/csv_upload.ts](bot/src/flows/csv_upload.ts).
