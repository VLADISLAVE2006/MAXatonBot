# API для бота MAX (регистрация пользователей)

Этот API обрабатывает регистрацию пользователей: сохранение согласия на обработку ПД и запись ФИО.

## Базовый URL

- Для локальной разработки: `http://localhost:8080`
- Когда поднимете на сервере — замените на реальный адрес (например, `https://api.ваш-домен.ru`)

## Аутентификация

**Все запросы (кроме `/health`) должны содержать заголовок:** X-API-Key: very-secret-key-2025


> ⚠️ Этот ключ совпадает с тем, что указан в `.env` на сервере. Убедись, что в вашем боте он тоже задан (например, в переменной окружения `API_KEY`).

## Проверка работоспособности (опционально)

**GET** `/health` — ответ `200 OK` с текстом `"OK"`.  
Этот эндпоинт публичный, помогает убедиться, что сервер жив.

## Эндпоинты

### 1. Сохранение согласия

**POST** `/api/user/consent`

**Заголовки:**
Content-Type: application/json

X-API-Key: very-secret-key-2025


**Тело (JSON):**
```json
{
  "user_id": 123456789,
  "agreed": true,
  "agreement_version": "1.0"
}
